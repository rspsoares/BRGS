Claro — vou te guiar de forma prática e completa para expor o SQL Server externamente **com segurança**, incluindo alteração de porta e firewall.

---

# 🧩 1. Ativar conexões remotas no SQL Server

1. Abra o **SQL Server Management Studio (SSMS)**
2. Conecte ao servidor
3. Clique com o botão direito no servidor → **Properties**
4. Vá em **Connections**
5. Marque:
   ✔️ *Allow remote connections to this server*

---

# 🌐 2. Ativar protocolo TCP/IP

1. Abra o **SQL Server Configuration Manager**

2. Vá em:
   **SQL Server Network Configuration → Protocols for [SUA INSTÂNCIA]**

3. Habilite:
   ✔️ **TCP/IP**

4. Clique duas vezes em **TCP/IP** → aba **IP Addresses**

---

# 🔢 3. Alterar a porta padrão (ex: 1433 → 21433)

Ainda dentro do TCP/IP:

1. Role até **IPAll**

2. Apague o valor de:

   * `TCP Dynamic Ports`

3. Em `TCP Port`, coloque por exemplo:
   👉 `21433`

4. Clique em **OK**

---

# 🔄 4. Reiniciar o serviço do SQL Server

No próprio Configuration Manager:

* Clique com o botão direito no serviço
* **Restart**

---

# 🔥 5. Liberar a porta no Firewall do Windows

Agora o mais importante 👇

1. Abra:
   **Windows Defender Firewall with Advanced Security**

2. Clique em:

   * **Inbound Rules → New Rule**

3. Escolha:

   * **Port**
   * TCP
   * Specific local ports: `21433`

4. Permita a conexão:
   ✔️ *Allow the connection*

5. Marque:
   ✔️ Domain
   ✔️ Private
   ✔️ Public (somente se necessário)

6. Nome:
   👉 `SQL Server Porta 21433`

---

# 🌍 6. Liberar no firewall externo / roteador

Se o servidor estiver atrás de NAT:

* Configure **Port Forwarding**

  * Porta externa: `21433`
  * IP interno: IP do servidor Windows
  * Porta interna: `21433`

---

# 🔐 7. Configurar autenticação SQL

Para acesso externo:

1. No SSMS:

   * Server Properties → **Security**

2. Marque:
   ✔️ *SQL Server and Windows Authentication mode*

3. Crie login:

   * Security → Logins → New Login

---

# 🧪 8. Testar conexão externa

Exemplo de string:

```
IP_DO_SERVIDOR,21433
```

Ou:

```
tcp:SEU_IP_PUBLICO,21433
```

---

# ⚠️ Boas práticas de segurança (importante)

Não pule isso 👇

* ❌ Evite expor diretamente à internet se possível
* ✔️ Use VPN (recomendado)
* ✔️ Restrinja IPs no firewall
* ✔️ Use senhas fortes
* ✔️ Considere criptografia (SSL)

---

# 🚀 Dica profissional

Se for ambiente de produção:

* Prefira acesso via **VPN**
* Ou use um **bastion host**
* Ou ainda Azure/AWS com regras de segurança
